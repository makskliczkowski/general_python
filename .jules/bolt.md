## 2025-11-01 - [Structure Factor Optimization]
**Learning:** In computing dynamical structure factors (and other spectral functions), transitioning to a Numba kernel over all eigenstates is efficient, but passing zero matrix elements (which represent forbidden transitions) wastes significant kernel execution time. Physical operators are typically sparse in the eigenbasis, so many transitions have zero weight.
**Action:** Filter out zero matrix elements (`> 1e-15`) and their corresponding energy differences *before* passing them to Numba kernels (`_structure_factor_kernel`). This early skipping mechanism can yield up to a 2-3x performance boost, especially for multi-initial-state finite-temperature evaluations.
## 2024-05-18 - Vectorize static susceptibility at zero temperature
**Learning:** In `physics/response/susceptibility.py`, replacing the Python iteration (`for n in range(1, N)`) with vectorized NumPy operations to calculate `chi_static` significantly improves performance (approx. 20x speedup). This is a classic bottleneck where Python loop overhead dominated array processing. However, vectorizing over the temperature dimension (`thermal_scan`) utilizing `np.outer` slowed down the code nearly 10x, primarily due to the massive intermediate allocations necessary for broadcasting causing L1/L2 cache misses.
**Action:** When working on numerical scientific Python logic, replacing dense loops with boolean masking and aggregated `np.sum` functions provides a massive reduction in iteration overhead. Be cautious of vectorizations that allocate significantly large dense intermediate arrays, as the memory latency often dwarfs any potential performance gains over simpler loops.
