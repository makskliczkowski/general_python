## 2025-11-01 - [Structure Factor Optimization]
**Learning:** In computing dynamical structure factors (and other spectral functions), transitioning to a Numba kernel over all eigenstates is efficient, but passing zero matrix elements (which represent forbidden transitions) wastes significant kernel execution time. Physical operators are typically sparse in the eigenbasis, so many transitions have zero weight.
**Action:** Filter out zero matrix elements (`> 1e-15`) and their corresponding energy differences *before* passing them to Numba kernels (`_structure_factor_kernel`). This early skipping mechanism can yield up to a 2-3x performance boost, especially for multi-initial-state finite-temperature evaluations.
## 2025-05-23 - [Vectorized Multi-Frequency Spectral Functions]
**Learning:** In computing multi-frequency spectral functions (`operator_spectral_function_multi_omega`, `susceptibility_bubble_multi_omega`), evaluating each frequency through a Python loop causes expensive and redundant matrix operations (e.g. $O(N^3)$ eigenbasis transformations) to execute iteratively. Standard numerical routines suffer immense overhead under this approach.
**Action:** Always hoist expensive invariant properties (basis transformations, occupations, thermal weights) outside of frequency loops and push the iteration grid into optimized C/C++ backends using NumPy broadcasting over `omegas[:, None]`. This refactoring typically changes $O(N_{\omega} \times N^3)$ to $O(N^3 + N_{\omega} \times N^2)$, reducing runtime by up to 30x.
