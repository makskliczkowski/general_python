## 2025-11-01 - [Structure Factor Optimization]
**Learning:** In computing dynamical structure factors (and other spectral functions), transitioning to a Numba kernel over all eigenstates is efficient, but passing zero matrix elements (which represent forbidden transitions) wastes significant kernel execution time. Physical operators are typically sparse in the eigenbasis, so many transitions have zero weight.
**Action:** Filter out zero matrix elements (`> 1e-15`) and their corresponding energy differences *before* passing them to Numba kernels (`_structure_factor_kernel`). This early skipping mechanism can yield up to a 2-3x performance boost, especially for multi-initial-state finite-temperature evaluations.
## 2025-11-01 - [Spectral Multi-Omega Vectorization]
**Learning:** In computing operator-projected spectral functions and susceptibilities for multiple frequencies, the previous implementation used a slow python loop over frequencies, calling a function that performs identical matrix element logic for every frequency.
**Action:** Vectorize array broadcasting over the frequency axis `omega` within the matrix element filtering logic. When evaluating `multi_omega` functions, broadcast frequency and perform operations such as `lorentzian` and array sum over the matrix element elements at once. This avoids repeated Python iterations and redundant operations, offering up to a 10x-20x performance improvement.
